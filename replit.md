# AURA Forge

AI-driven multi-chain smart-contract forge: describe a contract in plain English, get real Solidity or Rust/Anchor source generated by Claude, real `solc` compilation with an automatic self-healing retry loop on EVM, an LLM security audit, and real wallet-based testnet deployment (EVM via viem, Solana via Phantom devnet broadcast).

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — API server
- `pnpm --filter @workspace/aura-forge run dev` — frontend
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL`, `SESSION_SECRET`, `AI_INTEGRATIONS_ANTHROPIC_API_KEY`, `AI_INTEGRATIONS_ANTHROPIC_BASE_URL`

## Stack

- pnpm workspaces, Node.js, TypeScript
- Frontend: React + Vite + wouter + react-query + shadcn/Tailwind, `@monaco-editor/react` for code viewing, `viem` + `@solana/web3.js` for client-side wallet deployment
- API: Express 5, custom email/password auth (bcryptjs + express-session + connect-pg-simple)
- DB: PostgreSQL + Drizzle ORM
- LLM: Anthropic (Claude) via Replit AI Integrations (`lib/integrations-anthropic-ai`)
- EVM compilation: real, via the `solc` npm package
- API codegen: Orval (from OpenAPI spec) for REST endpoints; the forge job's live progress console uses a hand-rolled SSE endpoint (`GET /api/forge-contract/:id/stream`) not covered by the OpenAPI spec, since Server-Sent Events aren't representable in the codegen'd hook pattern

## Architecture notes

- Solana/Anchor compilation is **simulated and labeled as such**: the LLM writes real Anchor/Rust source plus a real IDL, but there is no cargo/Anchor toolchain in this environment, so no `.so` binary is actually built. Solana "deployment" is a real devnet transaction broadcast from the user's wallet, explicitly labeled as a verification broadcast rather than a full program deploy.
- EVM compilation is fully real: Claude generates Solidity, `solc` compiles it, and compiler errors are fed back into a self-healing LLM retry loop (up to 3 attempts) before failing the job.
- Generated contracts are constrained (via the LLM system prompt) to have no required constructor arguments, so the deploy UI never needs a dynamic constructor-argument form.

## User preferences

- Custom email/password auth was explicitly requested over Clerk/other third-party auth providers.
